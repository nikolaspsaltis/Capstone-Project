# Results Artifacts Guide

This document explains the evaluation artifacts stored in `results/`, how each is generated, and what metrics they contain.

## Current Artifact Types

| File pattern | Example file | Meaning | Generated by |
|---|---|---|---|
| `performance_*.csv` | `performance_20260304_134852.csv` | Per-request latency samples from the performance test | `python scripts/performance_test.py` (also called inside `run_capstone_evaluation.py`) |
| `evaluation_summary_*.json` | `evaluation_summary_20260302_182627.json` | Machine-readable script run summary (`script`, `exit_code`, `log_file`) | `python scripts/run_capstone_evaluation.py` |
| `evaluation_report_*.md` | `evaluation_report_20260302_182627.md` | Human-readable run summary by script | `python scripts/run_capstone_evaluation.py` |
| `*_*.log` (script logs) | `bruteforce_login_20260302_182627.log` | Raw stdout/stderr for each evaluation script | `python scripts/run_capstone_evaluation.py` |
| `graph_evaluation_diagnostics_*.png` | `graph_evaluation_diagnostics_20260302_182627.png` | Visual summary of script outcomes + failure root-cause classification | `python scripts/plot_evaluation_diagnostics.py --summary ...` |
| `graph_evaluation_exit_codes_*.png` | `graph_evaluation_exit_codes_20260302_182627.png` | Compatibility/legacy filename currently used to store the diagnostics-style outcome graph | created by copying diagnostics output when needed |
| `graph_latency_histogram_*.png` | `graph_latency_histogram_20260302_182451.png` | Archived latency distribution chart from an earlier run | historical artifact (kept for report appendix) |
| `graph_latency_timeseries_*.png` | `graph_latency_timeseries_20260302_182451.png` | Archived request-latency-over-time chart | historical artifact (kept for report appendix) |
| `graph_report_combined_2x2_*.png` | `graph_report_combined_2x2_20260302_182627.png` | Archived combined report-ready figure | historical artifact (kept for report appendix) |

## Metrics Captured

### `performance_*.csv`
- `request_index`: sample number
- `latency_ms`: request latency in milliseconds

From these samples you can derive:
- average latency
- median/p95/p99 latency
- min/max latency

`scripts/performance_test.py` also prints:
- average latency (ms)
- requests per second (throughput)

### `evaluation_summary_*.json`
- One record per script run:
  - `script`
  - `exit_code` (`0` pass, non-zero fail)
  - `log_file`

From this summary you can derive:
- script pass/fail rate
- script-level error rate (`failed_runs / total_runs`)

### `*_*.log`
- Raw run evidence from scripts
- Contains observed HTTP statuses and pass/fail messages
- Useful for diagnosing failures (for example connection, auth, or authorization issues)

### `graph_evaluation_diagnostics_*.png`
- Visual encoding of:
  - script outcomes (pass/fail via exit code)
  - categorized failure root cause extracted from logs

## Regenerate Results From Scratch

Run from repo root:

```bash
cd "/home/nikolas/Documents/UoE Documents/Captone-Project"
source secure-api-capstone/.venv/bin/activate
set -a
. ./.env
set +a
```

Terminal A (start API):

```bash
alembic upgrade head
uvicorn app.main:app --reload
```

Terminal B (generate artifacts):

```bash
# 1) Full evaluation bundle:
#    - evaluation_report_*.md
#    - evaluation_summary_*.json
#    - script logs (*_*.log)
#    - performance_*.csv (via performance_test.py in the runner)
python scripts/run_capstone_evaluation.py

# 2) Optional standalone performance CSV
python scripts/performance_test.py

# 3) Generate diagnostics graph from latest summary
LATEST_SUMMARY=$(ls -t results/evaluation_summary_*.json | head -n 1)
python scripts/plot_evaluation_diagnostics.py --summary "$LATEST_SUMMARY"
```

Optional compatibility copy (if your report expects the older filename):

```bash
TS=$(basename "$LATEST_SUMMARY" | sed -E 's/evaluation_summary_(.*)\\.json/\\1/')
cp "results/graph_evaluation_diagnostics_${TS}.png" \
   "results/graph_evaluation_exit_codes_${TS}.png"
```

## Notes
- If the API is not running, evaluation scripts may fail with connection errors; this will appear in logs and summary exit codes.
- `results/*.csv` and `results/*.log` are ignored by git to keep the repository lightweight; keep only selected artifacts you want to submit.
